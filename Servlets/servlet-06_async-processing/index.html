JAX-RS Asynchronous processing

Example implementation of JAX-RS asynchronous processing + client. This demo exposes two REST GET APIs.
Asynchrnonous: http://localhost:8080/wsasync/ws/data/async?delay=500
Synchrnonous: http://localhost:8080/wsasync/ws/data/sync?delay=500
On server, asynchronous call is processed by three threads, while synchonous call is processd by two threads. Notice JSON response: name - name of processing thread, startedBy: name of starting thread, triggeredBy: name of triggering thread. For client both calls, both calls are blocking (synchronous).

There are some URL parameters available for the service:
delay: delay between writes in millisecons

Notable java classes:
  • itx.asynctest.server.AsyncService - JAX-RS REST endpoint
  • itx.asynctest.server.DataService - Service generating data and running background jobs.
  • itx.asynctest.client.WSClient - Web Service client.
