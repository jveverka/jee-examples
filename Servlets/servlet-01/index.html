servlet-01-clients - simple http client

servlet-01-lab - This lab implements manual httpt session timeout control. ite.examples.servlet.session.SessionManager is singleton taking care of session management on application level. make use of http session management provided by servlet container.
deploy an application, check URL http://localhost:8080/servlet-01-lab/sessiontest - first time browser accesses the URL, you should see the "NEW registered session" message
- consequent URL access (browser reload) should display "Already registered session" message
- session invalidation timeout is set for 20 seconds

  1. remove/delete SessionManager class
  2. create new ite.examples.servlet.session.SessionAttribute class which implements HttpSessionBindingListener
  3. adapt private processRequest method of ite.examples.servlet.session.TestServlet, so it uses http session management provided by servlet container, check target implementation in servlet-01-solution project
  4. deploy the application, you behavior of application should remain same

servlet-01-solution - This lab utilizes httpt session timeout control provided by servlet container.
deploy an application, check URL http://localhost:8080/servlet-01-solution/sessiontest
- first time browser accesses the URL, you should see the "NEW registered session" message
- consequent URL access (browser reload) should display "Already registered session" message
- session invalidation timeout is set for 20 seconds
